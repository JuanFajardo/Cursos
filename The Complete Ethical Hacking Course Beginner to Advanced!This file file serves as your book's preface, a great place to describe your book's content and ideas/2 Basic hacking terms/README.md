**Basic hacking terms you will want to know getting started.**

1. Basic terminology such as white hat, grey hat, and black hat hacking.

   1. White Hat, enmarcado en la legalidad, con el consentimiento del objetivo \(https://goo.gl/iViMBX\).

   2. Grey Hat, abarca la legalidad como la ilegalidad.

   3. Black Hat, completamente en la ilegalidad, cada ves que se escucha HACKER se hace referencia a este grupo.

   4. Footprinting, es la recolección de información del objetivo\(puertos, protocolos, etc\), pero no solo abarca al medio digital también abarca el medio físico \(garbash\).

   5. Dos, Denial Of Service, puede ser realizado por una persona, con el objetivo de sobrecargar un servicio.

   6. DDoS, Distributed Denial Of Service, al igual que el anterior pero con muchas maquinas que son controladas gracias a un RAT el cual puede ser un FUD o terceros programas.

   7. FUD, Full Undetectable software que no es detectable por un antivirus.

   8. RAT, Remote Administration Tool, programa para controlar una maquina\(s\) de manera remota sin el consentimiento del usuario.

   9. Root Kit, software que se oculta del administrador de tareas, ejecutándose directamente en el kernel.

   10. Fising, clonando servicios, paginas, etc de tal manera que se pesque la información necesaria, útil al tener un DNS privado. 

   11. HTTP/HTPPS, Hypertext Transfer Protocol/Secure, un ataque de fising puede ser implementado fácilmente, darle SSL es un poco más difícil pero a la vez será mucho mejor ya que nadie se fija en las credenciales de seguridad.

2. Basic terminology including SQL injections, VPN , proxy, VPS, and key loggers.

   1. Sql injection, uno de los problemas más serios y más frecuentes.

   2. VPN, Virtual Private Network, nos permite tener un anonimato al momento de realizar un test.

   3. Proxy, otra manera de ser anónimo, es irreal lo que se muestra en el cine, usando 10 proxy cada proxy es lento, entre mas mucho  más lento, además la configuración es muy compleja.

   4. Tor, el ejemplo de un proxy, el cual nos permite un anonimato.

   5. VPS, Virtual Private Server, un servidor en la nube que puede usar como VPN y montar tu propio DNS. 

   6. Key loggers, un registro de lo que realiza un usuario en la maquina infectada.

   7. Terminal, la mejor manera de comunicarse con el SO\(GNU/Linux\), además de que es mucho as poderosa que la GUI, existen miles de comandos y solo con saber lo básico podemos ir aprendiendo con el transcurso del tiempo. 

   8. Firewall, software que analiza los paquetes de ingreso y salida, en la mayor parte primero debe vulnerarse el modem antes del host.

   9. Reverse shell, una manera de conseguir una terminal y con pequeños script montados en la nube o VPS, podemos sacar toda la información necesaria.
